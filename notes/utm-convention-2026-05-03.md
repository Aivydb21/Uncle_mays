# UTM Convention — Uncle May's Produce

**Effective:** 2026-05-03
**Owner:** CMO, with CRO + RevOps enforcement
**Why:** ML notebook on 2026-05-02 found UTM params missing on ~99% of conversion-attempt rows. Without consistent tagging on every paid URL, attribution stays broken regardless of how good the data plumbing is.

---

## The five params, in priority order

| Param | What it's for | Required on | Allowed values |
|---|---|---|---|
| `utm_source` | The platform or partner. | **All paid URLs** | `meta`, `google`, `tiktok`, `linkedin`, `youtube`, `email`, `sms`, `partner` (the partner's slug for partner referrals: `partner-rab`, `partner-1871`, etc.) |
| `utm_medium` | The format / channel type. | **All paid URLs** | `cpc` (paid search), `cpm` (display/awareness), `paid_social` (FB/IG feed/reels), `video` (YT/IG reels native), `cart_save` (our own save-cart email), `abandoned_cart`, `newsletter`, `transactional`, `organic_social`, `referral` |
| `utm_campaign` | The campaign objective + audience + month. | **All paid URLs** | `<objective>_<audience>_<yyyy_mm>`. Examples: `awareness_chicago_2026_05`, `conversion_retargeting_2026_05`, `pmax_lookalike_2026_05`, `fresh10_cold_2026_05` |
| `utm_content` | The variant — creative concept or ad set. | Recommended on paid (lets us A/B at variant level) | Free text, hyphen-separated. Examples: `vid_familytable_15s`, `static_kalebowl_a`, `static_kalebowl_b`, `text_savings_focus` |
| `utm_term` | Keyword / interest / placement. | Required on paid search; optional on social | Free text. For Google Ads, dynamic substitution: `{keyword}`. For Meta interests, the interest name. |

## Click-id capture (automatic, not in URL)

These are appended by the platform itself. Don't include them in the canonical campaign URL:

- `gclid` (Google Ads). Persisted to `localStorage["unc-gclid"]` on first hit so it survives long sessions.
- `fbclid` (Meta). Persisted to `localStorage["unc-fbclid"]`.

Both are read by `CheckoutClient` and forwarded to the Stripe PaymentIntent metadata, which the webhook then forwards to Meta CAPI / Google Ads Enhanced Conversions.

## Rules

1. **Lowercase, hyphen-separated, no spaces.** `awareness_chicago_2026_05` not `Awareness Chicago 2026-05`.
2. **Always include the date suffix in `utm_campaign`.** `_yyyy_mm` minimum so we can compare month-over-month performance without re-keying.
3. **No PII or session ids in URL params.** Use opaque slugs.
4. **Do not pass `?promo=CODE` in URL anymore.** That used to auto-apply codes; it no longer does (changed 2026-05-03 — customers must enter the code themselves in cart). The promo banner above the catalog still surfaces the code as a reminder if `?promo=` is set, but conversion stays the same.
5. **Test before launch.** Open the campaign URL in incognito, complete a $1 test order with the live promo code FRESH10 disabled, and verify the Stripe PaymentIntent metadata in the Dashboard contains all expected fields.

## How attribution flows end-to-end

1. Customer clicks a tagged ad: `?utm_source=meta&utm_campaign=awareness_chicago_2026_05`
2. `<UTMCapture />` (mounted in `src/app/layout.tsx:129`) runs `captureUTMParams()` from `src/lib/utm.ts` on first page load.
3. `sessionStorage["unclemays_utm"]` gets the full UTM bundle. `localStorage["unc-gclid"]` and `localStorage["unc-fbclid"]` get the click ids (longer TTL so they survive multi-day consideration windows).
4. Customer browses `/shop`, adds to cart, eventually starts checkout.
5. `CheckoutClient` calls `getUTMParams()` and reads the persisted gclid/fbclid + fbc/fbp cookies, then includes all of them in the body to `/api/checkout/intent`.
6. `/api/checkout/intent` writes them to Stripe PaymentIntent metadata.
7. On `payment_intent.succeeded`, the webhook fires Meta CAPI (with fbc/fbp/email hash/IP/UA) and GA4 Measurement Protocol (with transaction id + value).
8. The Stripe metadata + Meta CAPI events feed the ML pipeline at next ingest.

If any link in this chain breaks, UTMs end up null in the dataset. Today the chain is fully wired (as of 2026-05-03 commit). What's needed from the marketing team: **every campaign URL launched into Meta or Google Ads must follow the convention above.** No exceptions.

## Audit cadence

- **Weekly:** RevOps spot-check by sampling 10 random Stripe PaymentIntents from the past 7 days; >80% should have non-null `utm_source` for paid-traffic IPs. If lower, find which campaigns broke convention and tell the responsible owner.
- **Monthly:** rebuild the conversion-prediction parquet and re-check the column-missingness summary in the ML notebook. Goal: drop UTM-missing rate from ~99% (pre-launch baseline) to <30% within the first 60 days of paid spend.
