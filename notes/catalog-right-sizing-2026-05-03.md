# Catalog Right-Sizing — Default Increments by SKU (2026-05-03)

## Methodology & confidence

We have ~10 paid orders, so this is external research. Primary anchors: USDA ERS Food Availability per Capita (FADS) for annual per-person pounds, CSA box compositions (Main Street Farms, J&L Green Farm) which are explicitly designed around one-week household consumption for 2-3 people, and direct-to-consumer e-commerce (Misfits Market, Imperfect Foods) for how comparable operators package the same SKU. Per-capita pounds were converted to per-household-week using a 2.5-person-household and a 50% "weeks-purchased" factor for seasonal items (you don't buy asparagus 52 weeks/year). **Confidence is HIGH** for staples with strong USDA + CSA agreement (lettuce, carrots, potatoes, sweet potatoes, kale, chard), **MEDIUM** for items with sparser public data (ramps, sunchokes, daikon, microgreens), and **prices below are linear scales of the current per-lb price** — they are increment proposals, not pricing decisions.

## Notable changes (biggest deltas)

1. **Asparagus**: was 1 lb @ $9.38 → ½ lb @ $4.69 default. Modal home cook uses ½-1 lb; full pound sticker price was likely deterring add-to-cart. (CSA spring-share data)
2. **Ramps**: was 1 lb @ $17.50 → ¼ lb @ $4.38. A pound is a restaurant quantity; foragers and farmer's-market sellers price ramps in 4-8 oz portions. Drops the entry price 4×.
3. **Sweet potatoes**: was per-lb @ $5.63 → "each" (~½ lb) with default qty 2. Customers don't think in pounds for sweet potatoes; they think "2 for dinner, 4 for a casserole." Smalls SKU shifts to default 4. (Evolving Table; DTC norm)
4. **Tuscan kale, rainbow chard, mustard greens, alpine radishes**: all currently per-lb → switch to **bunch**. Every comparable e-comm sells these by the bunch, and one bunch = one household-week per CSA composition data. Big UX win, no real price change for chard/kale.
5. **Lettuces (green leaf, romaine, summer crisp)**: per-lb → **head**. CSA standard is "1 head = 3 servings = household-week." Per-lb pricing on a head of lettuce is a friction tax.
6. **Pastured whole chicken**: per-lb → **per bird**. DTC norm. Sticker is higher (~$39 vs $8.75) but matches how customers actually buy a whole chicken.
7. **Short ribs (bone-in & boneless) and lamb chops**: per-lb → **per-pack / per-pair**. Matches how meat is portioned for a single meal in every DTC butcher (Crowd Cow, Porter Road, Shepherd Song).

## Implementation notes (no code yet — schema/UI sketch)

To execute the above without breaking the existing per-lb data model, the catalog needs:

1. **Add `step` field to `CatalogItem`** (number, in current unit). Add-to-Cart adds `step` to qty, not 1. E.g. asparagus: `unit:"lb", step:0.5, defaultQty:1`. UI label renders as `"½ lb"` (computed: `step * unit`).
2. **Add `displayUnit` field** for items where the customer-facing unit differs from the inventory unit. E.g. tuscan kale stays `unit:"lb"` for inventory/COGS but `displayUnit:"bunch"` with `lbPerDisplay:0.45`. Cart line shows "1 bunch (~½ lb)."
3. **Add `unitType` enum**: `weight` (per-lb, with steps), `count` (per-each, e.g. sweet potatoes, daikon, whole chicken), `package` (clamshells, jars, bags — qty always integer ≥1, no step).
4. **Cart label rules**: For `weight` items, show `qty * step + " " + unit` ("2 × ½ lb = 1 lb"). For `count`, show `qty + " × " + name + " (~" + avgWeight + ")"`. For `package`, show `qty + " " + packDescription`.
5. **Default Add quantities** must be configurable per SKU (not a global default of 1) so that "smalls" defaults to 4, sweet potatoes default to 2, asparagus defaults to 1 (=½ lb), etc.
6. **Min/max qty per add** to prevent confusion: e.g., ramps min 1 (¼ lb), max 8 (2 lb). Optional but recommended.
7. **Price display**: keep the per-step price as the headline, with a small per-lb / per-each reference line below for shopper trust ("$4.69 / ½ lb · $9.38/lb").

No backend write needed today — this doc is the spec. Catalog migration should land behind a feature flag and be reversible per SKU.
